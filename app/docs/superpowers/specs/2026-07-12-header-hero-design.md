# LocalLink Header and Hero Design

## Objective

Create a client-ready first viewport that immediately communicates an international traveler forming a genuine connection with a Vietnamese local guide.

## Direction

Use a premium editorial travel marketplace composition: a restrained sticky header above a 55/45 text-to-image split. The content remains constrained to 1320px. The image fills a 4:5 panel with a 24px radius and contains only product-meaningful overlays.

## Header

- Height: 76px desktop, 68px mobile.
- Preserve existing routes for guide search, experiences, matching, dashboard, and guide onboarding.
- Keep language selection and mobile navigation functional and keyboard accessible.
- Use a subtle translucent surface and border only after scrolling.

## Hero

- Exact three-line headline: “Explore Vietnam / with someone who / truly gets you.”
- Concise supporting copy no wider than 540px.
- Primary CTA routes to `/match`; quieter secondary CTA routes to `/guides`.
- Compact trust signals sit directly beneath the CTA row.
- Local 4:5 photography fills the visual panel. Desktop overlays show match quality and guide/language/availability. Mobile retains only the compact guide overlay.
- No decorative cards, remote image dependencies, spellcheck-like underline, or search-form treatment.

## Responsive and accessibility

- Copy stacks before imagery below the desktop breakpoint.
- Faces remain visible through intentional object positioning.
- All controls provide visible focus, accurate labels, and 44px minimum touch targets.
- Reduced-motion preferences are respected.
- Verify at 390px, 1440px, 1920px, and 3840px with no horizontal overflow or console errors.

## Scope

This phase changes only the shared header, hero, relevant global tokens, and the hero asset. Lower homepage sections remain untouched until this first viewport passes visual verification.
